Find where your sources disagree.

Ordinary RAG answers a question from one retrieved document — so when two of your policies, specs, or contracts quietly contradict each other, it grounds in whichever one it happened to fetch and answers confidently wrong. Crosscheck does the inverse: it audits a whole body of knowledge and surfaces the silent conflicts — citing both sides, classifying the clash, and ranking by severity. Grounded by Microsoft Foundry IQ.

🔍
1 · Retrieve across sources

Foundry IQ pulls relevant passages from many documents at once, with citations.

⚖️
2 · Extract & compare claims

Each passage becomes atomic claims; the agent compares claims about the same subject.

📑
3 · Report cited conflicts

A ranked report — every conflict needs two citations from two different sources.

    Setup & access (Step 0)

    Go live on Foundry IQ — 3 commands
    1. Check access (read-only): ./infra/check_access.ps1
    2. Provision + upload the bundled corpus (an empty Azure is expected): ./infra/setup_foundry_iq.ps1
    3. Create the knowledge base in ai.azure.com → Build → Knowledge, set PROVIDER=foundry in .env, restart.

    Your Azure being empty is fine — Crosscheck brings its own data: the setup script uploads corpus/ for you. Full guide: infra/README.md.
